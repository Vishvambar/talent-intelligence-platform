# ADR 002: Regression over LambdaRank

## Decision
We use a standard Regression objective (Huber Loss) for the LightGBM/XGBoost ensemble rather than a pairwise/listwise ranking objective like LambdaRank.

## Context
Learning-to-Rank (LTR) traditionally uses LambdaRank to optimize for NDCG.

## Alternatives Considered
- **LambdaMART/LambdaRank**: Standard for search engines.
- **Pairwise Objective**: RankNet.

## Why Rejected
Our "labels" are not human-annotated 5-point scales (0=Irrelevant, 4=Perfect). They are continuous, high-fidelity synthetic scores (0.0 to 100.0) generated by the Phase 7 Teacher Ensemble. 
LambdaRank struggles to interpret continuous targets, often requiring arbitrary bucketing. Huber loss regression perfectly models the continuous target while remaining robust to the inevitable noise/outliers introduced by LLM teachers.
